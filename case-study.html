Case Study: Chat App

overview

The Chat App is a mobile application that allows users to engage in real-time conversations, send image and audio messages, and share their current location. This app allowed me to explore mobile development with React Native, focusing on key features such as real-time chat, multimedia sharing, and data accessibility.

challenge

To create a fully functional chat application for mobile devices, complete with user authentication, real-time data storage, and client-side data retention. This was my fifth project in the CareerFoundry program, providing a chance to deepen understanding of JavaScript applications and mobile app development.

objective

To build a native mobile chat application in React Native, meeting requirements for:

  • Real-time messaging with a robust chat interface
  • User authentication and data management
  • Accessibility for screen readers
  • Offline functionality to access conversations when disconnected

role

Full Stack Developer, with a focus on mobile development.

project scale

Completed in 21 days on a part-time basis. Required covering two learning milestones per week, which included:

  1. Building Native Applications with JavaScript
  2. Chat UIs & Accessibility
  3. Real-Time Applications & Data Storage
  4. Storing Data on the Client Side
  5. Communication Features

tools used

The Chat App was developed using the following technologies and assets:

  • React Native: For cross-platform mobile development
  • Expo: To streamline the development process and deploy on mobile
  • Firebase: For anonymous user authentication
  • Firestore: As a real-time database to store chat conversations
  • Gifted Chat: A library used to implement the chat interface and functionality
  • Design Assets: Provided by CareerFoundry to ensure alignment with specific UI guidelines

Approach

planning

To keep the process organized, I outlined key features and wrote user stories to keep the end user in mind.

Features

  • User Authentication: Allows users to authenticate anonymously and start chatting.
  • Chat Interface: Provides a simple, accessible interface for sending messages.
  • Image Sharing: Supports sending images from the gallery or via the camera.
  • Location Sharing: Allows users to share location data in a map view.
  • Customizable Chat Screen: Users can set their name and choose a background color.
  • Offline Functionality: Access previous messages even when offline.
  • Accessibility: Designed to work with screen readers for visually impaired users.

User Stories

  • As a user, I want to easily enter a chat room to start talking with friends and family.
  • As a user, I want to send messages to stay updated with friends and family.
  • As a user, I want to send images to share my current activities.
  • As a user, I want to share my location to show where I am.
  • As a user, I want to read my messages offline for easy reference.
  • As a visually impaired user, I want the app to be screen reader-compatible for accessibility.

development

Setup

Using Expo CLI, I initialized the React Native project and set up a virtual device to emulate the app. I then configured the emulator for testing layouts and styles using React Native's stylesheet.

Chat Interface

I integrated essential React Native components and utilized the Gifted Chat library, customizing it to be accessible and responsive. I also configured the app to detect different keyboard responses based on device types(iOS or Android).

Real-Time Data Storage

After exploring various options for real-time data solutions, I selected Cloud Firestore. To gain familiarity, I created a shopping-list app to implement Firebase for real-time data display and anonymous user authentication. I parallelled this experience to structure a database for the Chat App.

Client-Side Data Storage

To enhance offline accessibility, I implemented client-side data storage using AsyncStorage. I configured the app to detect network connectivity and conditionally render features based on connection status, allowing messages to persist offline.

Device Capabilities

Using Expo, I enabled access to the camera, microphone, and geolocation features. This allowed users to share multimedia and location data. Files were stored in Google Cloud Storage for persistence between sessions. I reviewed ethical and moral concerns surrounding privacy within the digital landscape and added prompts for consent to access.

Testing

I tested the app extensively on an emulator and with Expo Go on an iPhone 14. The final product was reviewed in two one-hour sessions with my mentor, where I received feedback on additional features and design improvements.

Summary

challenges faced

One of the most challenging aspects was implementing a redirect from shared location messages to the native maps application on users' devices. Without this redirect, location data could not be presented meaningfully within the app. By leveraging native maps on iOS and Android, I appended coordinate data to redirect links, allowing users to view shared locations in more detail.

what was learned

Building the Chat App allowed me to apply my knowledge of React Native and JavaScript to create a functional, real-time chat application. I gained a comprehensive understanding of mobile development and learned to evaluate optimal technologies for different use cases.

Overall, I achieved my original objective and am happy with the outcome. This hands-on experience has solidified my understanding of mobile app development and prepared me to take on more complex projects in the future.
